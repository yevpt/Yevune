# macOS 播放命令与独立队列窗口设计

- **日期**：2026-07-19
- **范围**：macOS 原生应用菜单、播放队列窗口
- **状态**：已实施

## 1. 目标

现代播放壳层已经有底栏、专注播放页、队列弹出框和迷你播放器，但队列只能从底栏进入，常用播放意图也没有出现在 macOS 应用菜单中。本切片增加原生“播放”菜单与独立队列窗口，让键盘用户和多窗口用户不必先返回曲库底栏。

## 2. 交互

- 应用菜单提供播放/暂停、上一首、下一首、随机、循环、显示播放队列和显示迷你播放器。
- 播放/暂停使用 `⌥⌘P`，上一首与下一首使用 `⌥⌘←` / `⌥⌘→`，避免无修饰空格干扰搜索和表单输入。
- 播放队列与迷你播放器分别使用 `⌥⌘Q` 和 `⌥⌘M` 打开。
- 菜单标题反映当前状态，例如“暂停”“关闭随机播放”“循环模式：单曲”。
- 空队列时禁用传输、随机和循环操作；两个窗口入口仍可用，并显示各自已有的空态。
- 独立队列窗口复用现有 `QueuePanel` 与同一个 `PlaybackController`，排序、删除和清空行为不复制。

## 3. 边界

- 不改变播放队列语义、AVFoundation 生命周期、媒体 URL 或系统媒体键。
- 不新增服务端、contract、Rust core 接口或第三方依赖。
- 不把队列重新放入专注播放页，继续遵守现代播放壳层的信息架构。

## 4. 测试与验收

- 纯策略测试覆盖空队列与非空队列的菜单可用性，以及播放、随机和循环状态标题。
- Swift 构建验证 `Commands`、窗口标识和 `PlaybackController` 接线。
- 真实窗口冒烟验证菜单打开独立队列和迷你播放器，关闭窗口不影响播放。
