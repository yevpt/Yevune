# ADR-0004：Garage 为唯一源，SQLite 本地 + 转码缓存入 Garage

**状态**：已接受（2026-07-10）

**背景**：存储用兼容 S3 的 Garage。需定"原始文件、索引、转码缓存"各放哪。

**决策**：
- **原始音频文件**：Garage 是唯一源（source of truth）。上传到 Garage，服务端从中读取。
- **SQLite 索引**：放服务器**本地磁盘**。
- **转码缓存**：放 **Garage**。

**理由**：
- 数据库跑在对象存储上性能极差 → 索引必须本地。
- 转码缓存体积大、可重建、访问模式适合对象存储 → 放 Garage，可安全按 LRU 淘汰。
- 入库靠主动扫描（列举 bucket 比对 ETag，增量处理）+ 客户端管理 API 双路径；Garage 的 bucket 事件通知能力有限，故用主动扫描而非事件驱动。

**后果**：扫描/转码读原文件走网络 IO（内网通常够快）；用 Range 读文件头解析标签以最小化传输。转码产物首次生成有延迟，之后命中缓存。
