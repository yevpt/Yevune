# ADR-0003：OpenSubsonic 兼容 + 自研扩展

**状态**：已接受（2026-07-10）

**背景**：客户端与服务端协议可自研或复用成熟协议。需要早期就能验证服务端。

**决策**：核心遵循 **OpenSubsonic** 协议（成熟、多用户原生、文档齐全），额外能力（多级歌单、库管理写接口、曲库访问控制、角色管理）走**命名空间隔离的自研扩展** `/rest/ext/*`，并通过 `getOpenSubsonicExtensions` 声明。

**理由**：
- 自研客户端就绪前，可直接用 Amperfy / play:Sub 等现成客户端验证服务端，相当于免费测试客户端。
- OpenSubsonic 原生多用户（每请求带 `u=用户名`），天然契合"家庭多用户 + 独立歌单空间"。
- 扩展隔离，既不破坏兼容，又能承载现代特性。

**后果**：须始终保持 OpenSubsonic 兼容（红线）。部分现代特性（如 HLS 自适应）协议支持有限，靠扩展补充。密码需可逆加密存储以支持 Subsonic token 认证（同 Navidrome）。
